COMMIT SU GIT

Il COMMIT è un istantanea del tuo progetto con un messaggio specifico che spiega che modifiche sono state apportate al progetto. È un punto di salvataggio.
SI PUò SEMPRE TORNARE AD UN COMMIT PRECEDENTE IN CASO DI ERRORI.

ALCUNI COMANDI:

  • GIT COMMIT -M "message"
    • Committare modifiche graduali

  • GIT COMMIT - A -M "message"
    • Eseguire il commit di tutte le modifichesaltando la fase di stagig

  • GIT LOG
    • Cronologia di tutte i commit

  • GIT ADD "nome file"
    • Traccia un nuovo file

  • GIT COMMIT --ALLOW-EMPTY -M "messaggio"
    • Mandare un commit vuoto, per esempio "start project"

  • GIT LOG -- STAT
    • Mostra tutte le modifiche dentro ai commit
